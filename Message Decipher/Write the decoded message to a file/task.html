Description

In this stage, your mission now shifts to preserving the valuable decoded messages by securely storing them in a new file called decoded-message.txt.

After successfully decoding the encrypted messages using the Atbash cipher, it's of utmost importance to protect the revealed information from falling into the wrong hands. To achieve this, we will harness the power of Node.js to create a classified repository.

Agent, your expertise in Node.js will be crucial as you complete this vital stage. As you write the decoded messages to a new file, remember the weight of the secrets you protect. The future of our mission depends on your ability to secure and preserve the decrypted information. You can replace the file if it already exists!

While keeping the old message after decoding the file, after writing the message to a new file, log this new message too: Finished writing the file

Stay vigilant, Agent, as you proceed with this critical phase of the project. The secrets are counting on you to keep them safe within the confines of the new file.

Objectives

In this stage, your program should:

  • Keep the previous decoding process without logging the result to the console.
  • Create a new text file called decoded-message.txt
  • Write the decoded message to the new file.
  • Log the message: Finished writing the file

Description

Welcome to the final stage, Agent! Your mission reaches its climax as you take command of the encryption process. In this stage, you'll harness the power of a custom shift cipher to encode the contents of the "message.txt" file and store them securely in a new file called "encoded-message.txt".

Using the sync-input library, you will prompt to enter the desired shift value through the console. Once you get the shift value, the program will read the contents of the "message.txt" file and encode each message using the specified shift cipher. The encoded messages will then be written to the "encoded-message.txt" file, ensuring their protection and accessibility for authorized individuals.

Here's an example of how the encoding process works with a shift value of 3, suppose we have the following message:

Hello, Agent.

To encode this message using a shift value of 3, each letter is shifted three positions to the right in the alphabet. For example, 'A' maps to 'D', 'B' maps to 'E', 'Z' maps to 'C', and so on. Applying the shift, the encoded message would be:

Khoor, Djhqw.

Remember you need to perform the previous operations first before taking the input. You can enter any input message you like and remember to convert the input to Number.

While keeping the old messages, after encoding the message, log this new message: Finished encoding the file and after writing: Finished writing the file

Good luck, Agent! The success of our mission rests in your hands, and protecting these encoded secrets is of utmost importance.

Objectives

In this stage, your program should:

  • Keep the previous decoding processes as is.

  • Take the input for the cipher shift.

  • Read the contents of message.txt file.

  • Encode the message.txt file with the new cipher.

  • Create a new text file called encoded-message.txt

  • Write the encoded message to the new file.

  • Log the message: Finished encoding the file

  • Log the message: Finished writing the file

The greater-than symbol followed by a space (> ) represents the user input. Note that it's not part of the input.
